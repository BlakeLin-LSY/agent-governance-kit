# skills/

**9 skills** — the curated subset (the live fleet has 53). These are the family
that tells the "judgment about what to leave out" story: the planning/alignment
lenses + the session lifecycle. A skill here is a composable prompt-procedure the
orchestrator invokes; together they form a reusable reasoning toolkit.

| Skill | What it does |
|---|---|
| `nucleus.md` | extract the irreducible core; classify each part NUCLEUS / SHELL / ACCIDENTAL |
| `reframe.md` | return to the true problem before rebuilding; generate alternative framings |
| `council.md` | multi-role debate (Architect · Skeptic · Pragmatist · …) to surface trade-offs |
| `step-back.md` | "less is more" alignment audit; strip work to nucleus, issue a pivot directive |
| `grill.md` | one-question-at-a-time interview to resolve the assumption stack before building |
| `guard-weekly.md` | orchestrates the deterministic health suite (`governance/guard.sh`) |
| `session-start.md` | cold-start orient + verifiable "Done means X" framing |
| `session-close.md` | pre-clear checklist incl. the memory-promotion gate |
| `evolve.md` | utilization-first self-evolution: audit skills/hooks against real usage |

The other 45 live in the private fleet. Shipping only these 9 is the point —
curation is the signal.
