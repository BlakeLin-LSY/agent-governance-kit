# Skill: grill

Resolve ambiguity before writing code, one question at a time.

## When to use
Before a new feature or an architectural change, whenever the request has more
than one valid interpretation.

## How it works
1. Ask **exactly one** question at a time — the highest-leverage unknown first.
2. Do not advance until the current answer is given and recorded.
3. Continue until the assumption stack is empty and "done means X" is defined.
4. Only then begin building.

## Output
A resolved assumption stack and a verifiable definition of done.

## Principle
A wrong assumption executed completely costs more than one clarifying question.
