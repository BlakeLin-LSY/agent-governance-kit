# Skill: reframe

Make sure you are solving the right problem before you solve it well.

## When to use
Before a major redesign, when a project feels off, or when effort keeps growing
without progress — a sign the framing, not the execution, is wrong.

## How it works
1. **State the problem as currently understood**, in one sentence.
2. **Audit the language** for smuggled assumptions and false constraints.
3. **Generate 3–5 alternative framings** of the same underlying need.
4. **Synthesize** one refined problem statement + the design implications it changes.

## Output
A single refined problem statement and the decisions it unlocks or rules out.

## Principle
A better question beats a better answer to the wrong question.
